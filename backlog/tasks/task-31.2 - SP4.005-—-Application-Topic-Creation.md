## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Create an Ansible playbook to provision application-specific Kafka topics for the web application. Define topics: app-messages (12 partitions, RF=3), app-events (6 partitions, RF=3), app-metrics (6 partitions, RF=3), and app-state (compacted, 6 partitions, RF=3). Set min.insync.replicas=2 on all topics.

Topics are created via kafka-topics CLI with SASL_SSL authentication using admin credentials (admin.properties generated by kafka-broker role). The playbook runs on a kafka_broker host (e.g., kb-01) since the kafka-topics CLI is available there.

This task can run in parallel with the Schema Registry and Connect role tasks (ordinal 4005 allows parallel execution alongside 4001/4002 since it only depends on TASK-30.2 which is Done).
